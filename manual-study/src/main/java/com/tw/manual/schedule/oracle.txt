        事件打包服务

    4.2 PL/SQL部分
        4.2.1正则表达式的改进
        4.2.2新sql语法
            在调用某一函数时，可以通过‘=>’来为特定的函数参数指定数据
            例 select f(x=>6) from dual;
        4.2.3内部单元内联
            类似C语言，用户通过内联函数或者宏实现使某些小的，被频繁调用的函数内联，编译后，调用内联
            函数的部分会被编译成内联函数的函数体，因而提高函数的效率，11g也可以同样实现内联函数。
        4.2.4序列的使用方法
            可使用 v_x :=seq_x.next_val 而不需要使用sql语句 select seq_x.next_val into v_x from dual
            为sequence赋值。
        4.2.5新的PL/SQL 数据类型
            引入新的数据类型 simple_integer, 比pls_integer 效率更高的整数数据类型。
        4.2.6continue 关键字
            通java 循环，略。
        4.2.7SUPER关键字
            在oracle对象类型中，通过super关键字实现继承性，类似java。
        4.2.8触发器
            触发器能力得到增强
        4.2.9对象依赖性改进
            oracle 11g 之前，如果有函数或者视图依赖于某张表，表结构如果发生变化，无论是否涉及到函
            数或者视图依赖的属性，都会使函数或者视图失效，oracle 11g之后，如果表改变的属性与相关的
            函数或者视图无关，则相关对象的状态不会发生变化。
        4.2.10细粒度权限控制
            之前oracle通过细粒度权限控制（FGAC）实现对数据库对象行级别的权限控制。现在，增加了对
            TCP包的FGAC安全控制。
        4.2.11结果集缓存(result set caching)
            大大提高程序性能。

        回滚段 用于存储用户数据被修改前的值
    5.7.区
        由物理上连续存在的块儿构成，是oracle存储分配的最小单位，由一个或多个块组成，一个或多个区组
        成段。当在数据库中创建带有实际存储结构的方案对象时，oracle将为该方案对象分配若干个区，以便
        组成一个对应的段来为该方案对象提供初始的存储空间。当段中分配的区都被写满后，oracle就为该段
        分配一个新区，以容纳更多的数据。
    5.8.数据块
        oracle 中最小的数据管理单位，也是执行输入输出操作时的最小单位。操作系统执行输入输出的最小单
        位是操作系统块，oracle块的大小是操作系统块的整数倍。最小为2K，最大可为64K。
        数据块可存储各种类型的数据，不论存储什么数据，每个数据块都有相同的结构：
            块头部 包含块的一般属性信息，如物理地址，所属段类型等。
            表目录 若块中存储的是表数据，则表目录存储关于该表的信息。
            行目录 存储该块中有效的行信息
            空闲空间 数据块中尚未使用的存储空间，当向数据中添加新数据时，将减少空闲空间。
            行空间 块中已使用的空间，存储了表或索引的数据。
            注：块头部，表目录，行目录共同组成了块的头部信息区。头部信息区并不存放实际的数据库数据，
            只起到引导系统读取数据的作用。因此，若是头部信息区损坏，则整个数据块损坏，其中存储的数据
            丢失。空闲空间和行空间共同组成块的存储区，两者总和就是块的总容量。
    总：表空间，段，区，数据块构成了oracle数据库的逻辑存储结构，可通过oracle数据库的数据字典进行查询
    ，逻辑存储结构从逻辑的角度分析数据库的组成，简单来说，多个数据块组成区，多个区组成段，多个段组成组
    成表空间，多个表空间组成数据库。
